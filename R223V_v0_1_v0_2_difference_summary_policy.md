# R223V v0.1 / v0.2 difference summary policy

## Policy

The comparison should default to a short difference summary before any detailed ledger view.

## Default comparison summary

Each sample should show:

- one sentence for v0.1 baseline limitation
- one sentence for v0.2 candidate improvement
- one sentence for hold risk

## Required comparison axes

- teacher default readability
- classroom event depth
- unit intensity effect
- review ledger completeness
- field leakage risk
- component safety
- sample contamination

## Recommendation

The future review-only pilot gate should not display long v0.1/v0.2 tables by default. It should use concise, teacher-readable summaries first, with deeper comparison available for reviewers.

## Hold condition

Hold if comparison text becomes schema explanation rather than teaching output review.
